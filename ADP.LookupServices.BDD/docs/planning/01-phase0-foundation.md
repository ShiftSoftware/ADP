# Phase 0: Foundation Refactoring

**Goal:** Fix structural problems in the existing BDD infrastructure and establish the shared test data directory. After this phase, the project is ready for environment-based testing and rapid expansion.

**Prerequisite:** None (first phase)

- The `AssertEntireDataSet()` method (lines 128-144)
- The Then steps that assert collection membership: `TheVehicleIsInVehicleEntries()`, `TheVehicleIsInSSCAffectedVINs()`, `TheVehicleIsInInitialOfficialVINs()`

Keep the Given steps and `WhenChecking()` — these are reused by other features.

### 10. Add Environment Loading Step

Add a Given step in `SharedStepDefinitions.cs` that loads an environment:

```csharp
[Given("the {string} environment is loaded")]
public void GivenTheEnvironmentIsLoaded(string environmentName)
{
    var path = Path.Combine(
        TestContext.GetTestDataRoot(),
        "environments", $"{environmentName}.json");

    var json = File.ReadAllText(path);
    var environment = JsonSerializer.Deserialize<TestEnvironment>(json);

    // Apply LookupOptions
    // Note: only simple properties here; resolver delegates stay as mocks
    _context.Options.WarrantyStartDateDefaultsToInvoiceDate =
        environment.LookupOptions.WarrantyStartDateDefaultsToInvoiceDate;
    // ... etc
}

[Given("loading vehicle {string} from the environment")]
public void GivenLoadingVehicleFromEnvironment(string vin)
{
    // Populates _context.Aggregate from the loaded environment's vehicle data for this VIN
    _context.Aggregate.VIN = vin;
    // ... populate VehicleEntries, SSCAffectedVINs, etc from environment
}
```

```gherkin

  A vehicle is considered "Authorized" or "Official" if it is imported by
```

## Risks and Considerations

- **Path resolution:** BDD tests need to find `ADP.TestData/` relative to the test project. Use a helper method that walks up from the test assembly directory to find the repo root.
- **JSON deserialization:** We use case-sensitive deserialization (default `System.Text.Json` behavior), so JSON property names must match C# property names exactly (PascalCase). This avoids the `id`/`ID` collision in Cosmos DB models and allows copy-pasting real data from Cosmos.
- **Existing inline Given steps stay:** The inline DataTable Given steps (`"a dealer with the following vehicles..."`) continue to work alongside environment loading. They're useful for focused edge-case scenarios.
